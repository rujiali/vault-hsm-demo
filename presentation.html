Key Protection

Where Does the Encryption Key Live?

Transit Key (AES-256)

Lives in Vault's storage — but stored as ciphertext. Without the HSM, it is unreadable.

HSM Master Key

Lives inside the HSM hardware — never exported. Used to wrap and unwrap the Transit key on demand.

The Result

Even if someone steals Vault's storage, they get encrypted keys they cannot unwrap — the HSM hardware is the only way to use them.

Key Lifecycle

Key Rotation Without Breaking Data

  • Keys are versioned — rotation creates a new version
  • New encryptions use the latest key version
  • Old ciphertext remains decryptable — no re-encryption required
  • Minimum decryption version can be set to retire old keys
  • Zero application downtime during rotation
# Rotate the key
vault write -f transit/keys/demo-key/rotate

# New ciphertext uses v2
vault:v2:XYZ789abc...

# Old v1 ciphertext still works
vault:v1:ABC123xyz... ✓
Key Rotation

How Apps Keep Working After Rotation

Rotation is Additive

A new key version is created — old versions are kept, not deleted. Vault maintains all versions until explicitly retired.

Ciphertext is Self-Describing

The key version is embedded in every ciphertext token. Vault reads it automatically and uses the correct version to decrypt.

vault:v1:ABC123...  ← uses v1
vault:v2:XYZ789...  ← uses v2

Zero App Changes

Apps send ciphertext to Vault — Vault selects the right key version transparently. No code change. No downtime.

Access Control

Separation of Duties

Different teams get different capabilities — enforced by Vault policy, not application logic.

Encrypt Only

Application can submit plaintext and receive ciphertext.


Cannot decrypt. Cannot manage keys.

Decrypt Only

Authorised service can decrypt ciphertext to retrieve data.


Cannot encrypt. Cannot manage keys.

Key Admin

Security team can rotate and manage keys.


Cannot encrypt or decrypt data.

Why This Matters

Meeting Your Compliance Obligations

🔐 Keys Protected by Certified Hardware

Encryption keys are stored and protected inside a Hardware Security Module — the industry standard for organisations that cannot afford key compromise.

📋 Complete Audit Trail

Every encryption and decryption operation is logged with identity, timestamp and context. You can prove who accessed what data, and when.

🔄 Keys Rotate Without Business Disruption

Encryption keys can be rotated on demand — existing data remains accessible, new data uses the latest key. No downtime, no re-engineering.

👥 No Single Person Holds All the Power

The team that encrypts data cannot decrypt it. The team that manages keys cannot read the data. Separation of duties enforced by policy, not trust.
